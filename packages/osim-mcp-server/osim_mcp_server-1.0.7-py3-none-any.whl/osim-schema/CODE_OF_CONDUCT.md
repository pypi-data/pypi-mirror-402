# 贡献者行为准则

## 我们的承诺

为了营造一个开放、友善的环境，我们作为贡献者和维护者承诺：无论年龄、体型、身体健全与否、民族、性征、性别认同与表达、经验水平、教育程度、社会地位、国籍、个人外貌、种族、宗教信仰、性倾向或其它特征，均以友善和尊重的态度参与本项目和社区。

## 我们的准则

有助于为本项目和我们社区创造积极环境的行为包括但不限于：

* **尊重和包容**：使用欢迎和包容的语言，尊重不同的观点和经历
* **建设性反馈**：优雅地接受建设性批评，专注于对社区最有利的事情
* **同理心**：在社区成员遇到困难时表现出同理心和善意
* **协作精神**：关注社区整体的利益，而不仅仅是个人利益
* **专业态度**：在专业和安全的环境中讨论技术问题

不可接受的行为包括但不限于：

* **骚扰言论**：使用性暗示、侮辱或贬损的言论，以及不当的关注或亲近
* **歧视行为**：基于个人特征的歧视或排斥，包括技术能力、工具偏好等
* **人身攻击**：发表侮辱性、贬低性或诋毁性评论，进行人身攻击或政治攻击
* **公开羞辱**：公开或私下羞辱他人，包括在代码审查中使用不恰当的语言
* **侵犯隐私**：未经授权发布他人的私人信息，如物理地址或电子地址
* **其他不专业行为**：在专业环境中不合适的其他行为

## 维护者的责任

项目维护者有责任为「可接受的行为」准则诠释和落实，并据实采取恰当、公平的矫正措施，以回应任何不可接受的行为。

项目维护者有权利及责任去删除、编辑、拒绝违背本行为标准的评论（comments）、提交（commits）、代码、wiki 编辑、问题（issues）等贡献，以及暂时或永久地封禁任何他们认为有不适当、威胁、冒犯、有害行为的贡献者。

## 适用范围

本行为标准适用于本项目所有平台，包括但不限于：
- 项目仓库、文档页面
- 项目邮件列表、讨论区、社交媒体账号
- 项目相关的线上或线下活动
- 在公共场合代表项目或社区时的行为

当某人代表本项目或其社区时，本行为标准亦适用于其行为。代表项目或社区的情形包括但不限于：使用项目的官方电子邮件地址、通过官方社交媒体账户发帖、在线上或线下活动中担任指定代表等。

## 监督落实

如发现滥用、骚扰或其他不可接受的行为，请通过以下方式联系项目团队：
- 发送邮件至：[项目维护团队邮箱]
- 在 GitHub 上联系核心维护者：[@维护者用户名]

所有投诉都将被及时、公正地审查和调查，并给出必要的回应。项目团队有义务对事件报告者保密。

## 处理流程

### 1. 举报流程
如果您是骚扰或不可接受行为的受害者或见证人，请：
1. 尽可能安全地脱离该情境
2. 保存相关证据（截图、日志等）
3. 通过上述联系方式向维护团队报告

### 2. 处理流程
维护团队接到报告后将：
1. **确认收到**：在24小时内确认收到报告
2. **调查评估**：由至少2名维护者进行调查评估
3. **制定方案**：根据情况严重程度制定处理方案
4. **执行措施**：执行相应的纠正措施
5. **结果通知**：向举报者通知处理结果（在保护隐私的前提下）

### 3. 处理措施
根据行为的严重程度，可能采取的措施包括：
- **警告**：对轻微不当行为的正式警告
- **暂时封禁**：在一定时间内禁止参与社区活动
- **永久封禁**：永久禁止参与项目所有活动
- **公开声明**：在严重情况下发布公开声明

## 具体行为指南

### 技术讨论准则
- **尊重技术多样性**：尊重他人选择的技术栈和工具偏好
- **对事不对人**：在代码审查和技术讨论中专注于技术本身
- **耐心指导**：对新贡献者保持耐心，提供建设性指导
- **承认错误**：勇于承认自己的错误，并从中学习

### 沟通准则
- **使用清晰语言**：避免使用可能被误解的讽刺或幽默
- **给予响应时间**：理解他人可能需要时间回复
- **尊重时间差异**：考虑贡献者所在的时区和文化背景
- **保持专业**：即使在激烈讨论中也要保持专业态度

### 安全研究准则
作为安全开源项目，我们特别强调：
- **负责任的披露**：遵循负责任的漏洞披露流程
- **尊重隐私**：在处理安全数据时严格遵守隐私规范
- **专业操守**：在安全研究中保持高度的专业和道德标准
- **合法合规**：确保所有安全研究活动合法合规

## 适用范围例外

技术批评本身，即使使用激烈的言辞，只要不涉及人身攻击，通常不被视为违反行为准则。技术辩论应基于事实和理性分析。
